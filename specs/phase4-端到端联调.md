## 3. F4.1 功能规格：LLM 意图解析

### 需求描述

优化角色提示词，使 LLM 能从自然语言中精确提取抓取目标和操作类型。

### 意图分类

| 意图类别 | 自然语言示例 | 映射动作 |
|----------|-------------|----------|
| 抓取特定物体 | "把那杯水拿过来" | photo + vla_grasp("water cup") |
| 抓取颜色物体 | "抓取红色方块" | photo + vla_grasp("red block") |
| 抓取位置物体 | "左边那个拿过来" | photo + vla_grasp("left object") → 需多轮确认 |
| 放下物体 | "把东西放下" | move_joints(放下位置) + gripper(open) |
| 归位 | "回到初始位置" | move_joints(零位) |
| 状态查询 | "机械臂能动吗" | get_status |

### 提示词追加内容

```yaml
## 抓取指令处理
当用户要求抓取物体时：
1. 如果用户指定了明确目标（颜色/形状/名称），直接调用 vla_grasp
2. 如果用户说"那个/这个/左边那个"，先确认目标物体
3. 抓取前告知用户"我来帮你抓取[目标]"
4. 抓取完成后报告结果"已经抓到了[目标]，要我放哪里呢？"
```

### 验收标准

1. "抓取红色方块" → LLM 调用 `vla_grasp("红色方块")`
2. "把左边的积木拿过来" → LLM 触发多轮确认("请问左边的积木是什么颜色？")

## 4. F4.2 功能规格：多轮交互

### 需求描述

实现对话状态机，支持指代消解、确认机制和多步操作。

### 对话状态机

```
IDLE ──(用户:"抓取左边那个")──→ AWAIT_CONFIRMATION
  │                                   │
  │                            (用户:"红色那个")
  │                                   ↓
  │                            执行 vla_grasp("red")
  │                                   │
  │                            (成功) (失败)
  │                              ↓      ↓
  └────("抓取完成")──── IDLE    IDLE("抱歉没抓到，再试一次？")
```

### 关键交互模式

| 模式 | 触发条件 | 响应 |
|------|----------|------|
| 指代消解 | 用户说"那个/这个" | "请问你指的是哪个？什么颜色的？" |
| 确认机制 | 抓取可能出错 | "确认要抓取[目标]吗？" |
| 连续操作 | "抓过来然后放下" | 依次执行 vla_grasp + move_joints + gripper |
| 失败重试 | VLA 置信度 < 0.5 | "我没太看清，能不能换个角度？" |

### 验收标准

1. 指代不明时，LLM 主动问澄清问题
2. 多步操作能正确分解为工具调用序列
3. 抓取失败后自动提示重试

## 5. F4.3 功能规格：异常处理

### 需求描述

覆盖抓取全链路的异常场景，实现优雅降级。

### 异常场景矩阵

| 异常 | 检测方式 | 处理策略 |
|------|----------|----------|
| 摄像头故障 | `camera.take_photo` 返回 error | "抱歉，我的摄像头好像出了点问题" |
| 照片过暗 | 图像平均亮度 < 阈值 | "光线有点暗，能开个灯吗？" |
| VLA 服务超时 | HTTP 请求 > 30s | "推理服务有点慢，请稍等或稍后重试" |
| VLA 置信度过低 | confidence < 0.3 | "我不太确定抓取方式，换个角度试试？" |
| 机械臂通信超时 | UART 无回传 > 5s | "机械臂没有响应，请检查连接" |
| 关节角度超限 | angles 超出 0-180° | 裁剪到安全范围，警告用户 |
| 抓取失败 | gripper 闭合后物体滑落 | "好像没抓住，我再试一次"（最多 3 次） |

### 安全限位（ESP32 固件端）

在 `xiaozhi_custom_board.cc` 的 `robot.arm.move_joints` 回调中增加：

```cpp
// 关节角度安全裁剪
for (int i = 0; i < 6; i++) {
    if (angles[i] < JOINT_MIN[i]) angles[i] = JOINT_MIN[i];
    if (angles[i] > JOINT_MAX[i]) angles[i] = JOINT_MAX[i];
}
```

### 验收标准

1. 每个异常场景都有对应的用户友好提示
2. 安全限位代码在编译后验证

## 6. F4.4 功能规格：时延优化

### 需求描述

优化全链路时延，目标从预估的 3-5 秒优化到 < 5 秒稳定。

### 时延分解

| 环节 | 当前预估 | 优化目标 | 优化手段 |
|------|----------|----------|----------|
| 语音/文字识别 | 200ms | 200ms | — |
| LLM 意图解析 | 500-1000ms | 500ms | 缩短提示词、使用 deepseek-chat |
| 摄像头拍照 | 300-500ms | 300ms | 降低分辨率到 320×240 |
| 图片传输 | 200-500ms | 200ms | JPEG 压缩质量 70% |
| VLA 推理 | 100-300ms (GPU) | 150ms | FP16 推理、TensorRT |
| 指令下发 | 100ms | 100ms | — |
| 机械臂执行 | 500-2000ms | 500ms / step | 提高移动速度 |
| **总计** | **~3-5s** | **< 5s** | — |

### 验收标准

1. 端到端时延（从指令发送到机械臂开始动作）< 5 秒
2. 文字消息交互场景下，回复时延 < 2 秒
