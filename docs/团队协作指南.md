# 团队协作指南

> 本文档面向所有小组成员，即使你不熟悉 Git 和 GitHub，按照步骤操作即可。

---

## 一、仓库权限设置（组长操作）

**1. 添加小组成员到仓库**

- 打开仓库主页：https://github.com/Re02VClife/desk-robot
- 点击 **Settings** → **Collaborators** → **Add people**
- 输入成员的 GitHub 用户名或邮箱，选择角色为 **Write**（可推送代码）
- 成员会收到邀请邮件，接受后即可访问

**2. 分支保护（防止意外覆盖）**

建议在 **Settings → Branches → Add rule** 中设置：
- **Branch name pattern**: `master`
- 勾选：`Require a pull request before merging`
- 勾选：`Require approvals` (至少 1 人审查)

这样可以防止直接推送 master，必须通过 Pull Request 合并。

## 三、新手教程：拉取代码

### 3.1 安装 Git

- **Windows**: 下载 https://git-scm.com/download/win → 安装（全部默认选项）
- **macOS**: 打开终端，运行 `brew install git` 或 `xcode-select --install`
- **Linux**: `sudo apt install git`

### 3.2 配置 Git

打开终端（Windows 右键 → **Git Bash Here**），运行以下命令，**替换成你的信息**：

```bash
git config --global user.name "你的名字"
git config --global user.email "你的邮箱@example.com"
```

> 名字用真实姓名，方便识别是谁提交的代码。

### 3.3 克隆代码

```bash
# 方式一：HTTPS（推荐新手，需输入 GitHub 账号密码）
git clone https://github.com/Re02VClife/desk-robot.git

# 方式二：SSH（需配置 SSH Key）
git clone git@github.com:Re02VClife/desk-robot.git
```

克隆完成后：

```bash
cd desk-robot
```

## 四、协作工作流

### 核心原则：**每个人在自己的分支上开发，互不干扰**

```
      master（稳定版，只合并经过审查的代码）
         ↑
    feature/xxx（你的开发分支，随意修改）
         ↑
      本地代码（你电脑上的文件）
```

### 4.1 创建自己的分支

**每个人只做一次**：创建属于自己的开发分支。

```bash
# 以你的名字或模块名命名分支
git checkout -b feature/你的名字

# 推送到远程（让其他人也能看到）
git push -u origin feature/你的名字
```

例如：`feature/张三` 或 `feature/音频模块`

### 4.2 日常开发流程

每次开始工作前：

```bash
# 第1步：切换到 master 拉取最新代码
git checkout master
git pull

# 第2步：切换到自己的分支，并把 master 的最新代码合并进来
git checkout feature/你的名字
git merge master
```

**修改文件 → 提交 → 推送**：

```bash
# 第3步：查看改了什么
git status

# 第4步：暂存修改
git add 你修改的文件名
# 或者暂存所有改动：git add .

# 第5步：提交到本地
git commit -m "简要说明改了啥"

# 第6步：推送到远程（自己的分支）
git push
```

> **小提示**：建议每完成一个小功能就提交一次，不要攒了一周才提交。

### 4.3 怎么把代码合并到 master

当你的功能开发完成、测试通过后：

**方式一：通过 GitHub 网页（推荐）**

1. 打开 https://github.com/Re02VClife/desk-robot
2. 点击 **Pull requests** → **New pull request**
3. `base: master` ← `compare: feature/你的名字`
4. 填写说明 → **Create pull request**
5. 通知组长或同事审查代码
6. 审查通过后点击 **Merge pull request**

**方式二：命令行**

```bash
git checkout master
git pull
git merge feature/你的名字
git push
```

### 4.4 拉取别人的最新代码

如果你需要获取其他成员的最新改动：

```bash
git checkout master
git pull                     # 拉取 master 最新
git checkout feature/你的名字
git merge master             # 合并到你的分支
```

## 五、常见问题

### Q：不小心把代码提交到了 master 分支？

```bash
# 撤销上一次提交（但保留修改内容）
git reset --soft HEAD~1
# 创建自己的分支并切换过去
git checkout -b feature/你的名字
git add .
git commit -m "说明"
git push -u origin feature/你的名字
```

### Q：推送时提示冲突（Conflict）？

```
# 多人改了同一个文件会导致冲突
# 先拉取最新代码
git pull

# Git 会标记冲突文件，打开它们
# 找到类似下面的内容：
# <<<<<<< HEAD
# 其他人的代码
# =======
# 你的代码
# >>>>>>> feature/xxx

# 手动保留正确的版本，删除标记符号
# 然后重新提交
git add 已解决冲突的文件
git commit -m "解决冲突"
git push
```

### Q：怎么查看所有人的分支？

```bash
git branch -a              # 查看所有分支
git checkout feature/某人  # 切换到别人的分支看看
git checkout feature/你的名字 # 切回自己的分支
```

### Q：不想保留某个修改？

```bash
# 丢弃单个文件的所有修改
git checkout -- 文件名

# 丢弃所有修改（谨慎操作！）
git stash
```

## 七、总结

```
每日工作流程：

1. git checkout master && git pull         ← 拉取最新
2. git checkout feature/你的名字              ← 切到自己的分支
3. git merge master                         ← 同步最新
4. 修改代码...                              ← 干活
5. git add . && git commit -m "说明"         ← 提交
6. git push                                 ← 推送
7. 功能完成 → 网页上提 Pull Request → 等审查 → 合并到 master
```

**牢记**：只在 `feature/你的名字` 分支上修改代码，绝不直接在 `master` 上改。
