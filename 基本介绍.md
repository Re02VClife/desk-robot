# 小智开发板 — 自定义主板（ESP32-S3-WROOM-1-N16R8）

## 1. 硬件概要

### 主控

- **芯片**：ESP32-S3-WROOM-1-N16R8（乐鑫）
- **Flash**：16MB（内置）
- **PSRAM**：8MB（内置，Octal SPI）
- **封装**：SMD 18×25.5mm，41引脚

### 供电

- **输入**：3.3V / 5V（外置电源模块，电池经升压板至5V）
- **LDO**：ME6211C33M5G-N（5V→3.3V，SOT-23-5）
- **5V升压板**：U9，电池升压至5V

### 烧录与调试

- **串口芯片**：CH340（外部模块），RX/TX/GND
- **调试引脚**：IO43(TXD0)、IO44(RXD0)，通过 JP1（6P 2.54排母）引出
- **初次烧录**：通过 CH340 串口烧录
- **后续升级**：WiFi OTA 更新

## 2. 完整引脚映射

### 音频

| 引脚 | 功能        | 说明                                |
| ---- | ----------- | ----------------------------------- |
| IO4  | 麦克风 LRC  | ZTS6672 左右时钟（I2S）             |
| IO5  | 麦克风 DIN  | ZTS6672 数据输出（I2S）             |
| IO6  | 麦克风 BCLK | ZTS6672 位时钟（I2S）               |
| IO7  | 功放 DIN    | CH98357 / MAX98357A 数据输入（I2S） |

> **注意**：功放 BCLK 和 LRCK 与麦克风共享 IO6/IO4（同一条 I2S 总线）。功放 SD 引脚通过 R1（1MΩ）上拉到 5V 硬件常开，无需 GPIO 控制。

### 显示 — ST7789 SPI（U7 主连接器）

| 引脚 | 功能 | 说明             |
| ---- | ---- | ---------------- |
| IO21 | SCL  | SPI 时钟         |
| IO47 | SDA  | SPI 数据（MOSI） |
| IO45 | RES  | 复位             |
| IO40 | DC   | 数据/命令选择    |
| IO41 | CS   | 片选             |
| IO42 | BLK  | 背光（PWM 控制） |

> **⚠️ 重要**：板上有两个显示连接器（U7 和 8pin），**必须使用 U7**。8pin 连接器的 IO11/IO12 与 MPU-6500 的 I2C 冲突，不要同时焊接。

### 姿态传感器 — MPU-6500

| 引脚 | 功能 | 说明                    |
| ---- | ---- | ----------------------- |
| IO11 | SDA  | I2C 数据                |
| IO12 | SCL  | I2C 时钟                |
| —   | AD0  | 接 GND，I2C 地址 = 0x68 |

### LED 与按键

| 引脚 | 功能      | 说明                                         |
| ---- | --------- | -------------------------------------------- |
| IO48 | RGB LED   | WS2812B（XL-5050RGBC）                       |
| IO0  | BOOT 按键 | 短按：切换对话状态；启动时按住：进入配网模式 |
| IO38 | VOL+ 按键 | 音量增大                                     |
| IO39 | VOL- 按键 | 音量减小                                     |

### 电池电量检测

| 引脚 | 功能     | 说明                                     |
| ---- | -------- | ---------------------------------------- |
| IO2  | ADC1_CH1 | 电池电压检测（100k+100k 分压，比例 0.5） |

- 分压电阻：R3（100k，上臂接5V）、R4（100k，下臂接GND）
- 满电 4.2V → 分压 2.1V（在 ADC 量程 0~3.1V 内）

### 机械臂控制 — UART1（直连 SO101 舵机总线）

| 引脚 | 功能 | 说明 |
|------|------|------|
| IO9  | TX   | → Bus Servo Adapter R（标准交叉接法） |
| IO10 | RX   | ← Bus Servo Adapter T |
| 3.3V | VCC  | 驱动板逻辑供电（不可省略） |
| GND  | G    | 共地 |

- **协议**：Feetech SCServo 二进制协议（1,000,000 bps）
- **舵机**：6× STS3215（型号 252），ID 1-6
- **驱动板**：Bus Servo Adapter V1.1，跳线帽断开（UART 模式）

### 触摸传感器

| 引脚 | 功能 | 说明 |
|------|------|------|
| IO3 | 触摸输入 | 开关式触摸传感器，触发「被摸头」交互 |

### 扩展接口

| 接口            | 引脚                   | 说明                          |
| --------------- | ---------------------- | ----------------------------- |
| CN1（4P 1.0mm） | IO17, IO18, IO19, IO20 | 扩展接口 1                    |
| CN2（4P 1.0mm） | IO8, IO37, IO42, IO1   | 扩展接口 2（IO42 与背光共享） |

### 空闲引脚

IO13、IO14、IO15、IO16 — 可用于后续扩展（如摄像头）。

## 4. 软件环境

### 框架与工具

- **ESP-IDF**：v5.5.3（安装路径：`E:\esp\Espressif\frameworks\esp-idf-v5.5.3`）
- **编译工具**：VSCode + ESP-IDF 扩展
- **项目**：xiaozhi-esp32（小智 AI 语音助手）v2.2.4

### 板级配置

- **目录**：`main/boards/xiaozhi-custom/`

## 5. 编译与烧录

### 首次编译

```bash
# 在 VSCode ESP-IDF Terminal 中
cd xiaozhiAI
idf.py set-target esp32s3
idf.py menuconfig
# 选择: Xiaozhi Assistant → Board Type → XiaoZhi Custom PCB
# 确认: Serial flasher config → Flash size = 16MB
idf.py build
```

### 烧录（通过 CH340，连接 IO43/IO44）

```bash
idf.py -p COMx flash monitor
```

### 手动进入下载模式

按住 BOOT(IO0) → 按 EN → 松开 EN → 松开 BOOT

### OTA 更新

首次烧录后，后续固件通过 WiFi OTA 自动更新（双分区回滚）。

## 6. 注意事项

1. **显示连接器**：必须使用 U7（主连接器），不要焊接 8pin 连接器，否则 IO11/IO12 引脚冲突
2. **功放 SD 引脚**：硬件常开（R1=1MΩ 上拉至 5V），软件无法关断功放
3. **麦克风驱动**：ZTS6672 替代 INMP441，I2S 时序兼容，但需验证实际表现
4. **MPU-6500 驱动**：小智项目未内置 MPU-6500 驱动，需自行添加或通过 MCP Server 集成
5. **CN2 的 IO42**：与显示屏背光共用，使用 CN2 时背光将受影响

## 7. 运动模块 — SO101 6-DOF 机械臂 ✅

通过 UART1（IO9/IO10）直连 SO101 舵机总线，无需额外 MCU。已实现：

- **SCServo 协议直驱**：ESP32 直接生成 Feetech 二进制指令，1M bps
- **MCP 工具**：6 个控臂工具（move_joints/gripper/home/stop/get_status）
- **LLM 直控**：deepseek-v4-flash 自由调用 MCP 工具编排动作
- **多步指令**：支持"X 然后 Y 两秒后 Z"自然语言格式
- **安全约束**：固件安全锁 + 限位裁剪
- **触摸交互**：IO3 传感器触发随机动作反应
- **摄像头**：USB 摄像头（PC 端 OpenCV 抓图 → LLM 看图）
